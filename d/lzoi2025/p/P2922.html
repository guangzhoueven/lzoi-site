[USACO08DEC] Secret Message G

题目描述

贝茜正在领导奶牛们逃跑。为了联络，奶牛们互相发送秘密信息。

信息是二进制的，共有 MMM（1≤M≤500001 \le M \le 500001≤M≤50000）条，反间谍能力很强的约翰已经部分拦截了这些信息，知道了第 iii 条二进制信息的前 bib_ibi​（1≤bi≤100001 \le b_i \le 100001≤bi​≤10000）位，他同时知道，奶牛使用 NNN（1≤N≤500001 \le N \le 500001≤N≤50000）条暗号．但是，他仅仅知道第 jjj 条暗号的前 cjc_jcj​（1≤cj≤100001 \le c_j \le 100001≤cj​≤10000）位。

对于每条暗号 jjj，他想知道有多少截得的信息能够和它匹配。也就是说，有多少信息和这条暗号有着相同的前缀。当然，这个前缀长度必须等于暗号和那条信息长度的较小者。

在输入文件中，位的总数（即 ∑bi+∑ci\sum b_i + \sum c_i∑bi​+∑ci​）不会超过 5×1055 \times 10^55×105。

输入格式

第 111 行：两个整数 MMM 和 NNN。

接下来 MMM 行，其中的第 iii 行：一个整数 bib_ibi​，后跟 bib_ibi​ 个空格分隔的“0”和“1”，描述截取的消息 iii。

接下来 NNN 行，其中的第 jjj 行：一个整数 cjc_jcj​，后跟 cjc_jcj​ 个空格分隔的“0”和“1”，描述暗号 jjj。

输出格式

NNN 行，第 iii 行表示第 iii 个暗号可以匹配的消息数。

4 5 
3 0 1 0 
1 1

3 1 1 0

1 1

提示

四条消息；五条暗号。截获的消息以 010、1、100 和 110 开头。可能的暗号以 0、1、01、01001 和 11 开头。

0：匹配 010: 111 个匹配。

1：匹配 1、100 和 110，333 个匹配。

01：匹配 010，111 个匹配。

01001：匹配 010，111 个匹配。

11： 匹配 1 和 110，222 个匹配。
